Automatic backup of Qiqqa

Is it possible to do an automatic backup of my Qiqqa data at a present time of day? How do you choose the directory you wish Qiqqa to automatically back up to?
Thank you, Tanya Nix

    Hello Tanya,

    Qiqqa can do an on-line storage "cloud" backup (also very convenient if you want to use multiple computers, or share libraries), or backup to a local directory on your computer.

    Both a sync, and a local back are quite easy to do, but (as far as I know) both require your action to initiate a backup.

    For local backup of ALL LIBRARIES:
    * In HOME TAB, select TOOLS, then select BACKUP, and then select the folder you wish to backup to.

    For local backup of a SINGLE LIBRARY:
    * in your LIBRARY tab (where "LIBRARY" is whatever you named the library) , select "Export Library", then select "Export Folder", and then select the folder you wish to backup to.

    For a sync (a storing of backup to the "cloud") of ALL LIBRARIES:
    * in your HOME TAB, select LIBRARIES, and then SYNC LIBRARIES

    For a sync (a storing of backup to the "cloud") of a SINGLE LIBRARY:
    * in your LIBRARY (where "LIBRARY" is whatever you named the library) select SYN LIBRARY.

    ALSO - any editing that you may do to your library (adding sources, annotations etc.) is being automatically stored on your local drive. It's not like a word process, where you have to constantly save. This data is stored in the "AppData" data folder... but you probably don't want to mess with it directly.

    PS
    I'm not associated with Qiqqa, other than through being a general user of their software.

    Tanya - the options above are all good. Syncing is probably the easiest option for you as it is just one button click every now and then.

    Alternatively, if you have backup software, point it at the Qiqqa data folder. To find the Qiqqa data folder open the config screen and press the "open data folder" button...

    NB: Do not put the data folder under a DropBox account, as DropBox is known to corrupt application database files if they are changed in two places simultaneously!
