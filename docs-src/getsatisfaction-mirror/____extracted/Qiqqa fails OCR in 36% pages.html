Qiqqa fails OCR in 36% pages

Hi, thanks in advance.
I've just discovered Qiqqa. It's perfect for me and I want it to work the way I expect, but, by now...
OCR fails in 1374 pages of 3785 total. 36% fail.
I cannot rely on Qiqqa this way.
I've re-OCRed documents with Acrobat and now, even in those PDFs that were just pictures, text is perfectly selectable on Acrobat Pro XI.
In spite of that, Qiqqa doesn't take advantage of it.
It seems Qiqqa needs to populate files in the OCR location folder with words got by itself using its own character recognition capabilities.
Why ? Acrobat has done the job before !! Qiqqa should only load the words into its database and index them !!
I hope you could find me a solution.
I've found no other software but Qiqqa that let me bring order to such a chaos of PDFs.
I'll send you an email with a PDF that Qiqqa OCRs partially... many pages fail to be OCRed.
In addition, Qiqqa user interface should highlight documents with pages that failed to be OCRed.
It would be good for people to trust Qiqqa and play it safe at least.
Once again... Thank you.

    Dennis over 5 years ago
    When I import files that have been scanned by my Scansnap scanner and OCR'd by ABBY Finereader, the files that appear in the OCR directory are the OCR text extracted from the PDF. Qiqqa's OCR never runs character recognition, it simply pulls out existing text.

    Though I do routinely OCR my files before importing them into Qiqqa, I do agree that when I did try the built in OCR a few years ago, it had issues. Recognizing two column pages as a single column was a problem. I also had issues with some unrecognized pages.

    Walter over 5 years ago
    Hey Dennis,

    Thanks for your reply.

    Finally, I've found that indexing capabilities of Windows itself are pretty good.
    Windows can index pdf contents by means of a plugin called iFilter.
    Once the index is built you can use Windows Desktop Search, even in advanced mode, and find the pdfs that contain the words you've typed, by a particular author, or configure an even more refined search.

    I've also found a free software, PDF-ShellTools, that embeds its functionality into the shell of Windows.
    When you right-click on any pdf file in Windows Explorer you get options to modify pdf metadata, bulk rename files, run scripts...
    Also an InfoEdit tab is added to the file properties dialog and you can change pdf metadata just from within this tab.
    But what I've liked the most is that you can create your own pdf metadata fields and map them to any of the windows file metadata fields so you can show them in Windows Explorer and use them to sort and filter the file list.

    I'll try to organize all my pdf stuff this way, updating pdf metadata from within Windows Explorer, and maybe then I'll go back to Qiqqa to take profit of its visual interface.

    Thanks again for you interest.

    Walter.
