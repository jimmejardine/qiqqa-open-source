Paul Moody over 9 years ago

Programming API

Hi Qiqqa Team,

Are you planning on publishing an API? I can imagine using qiqqa as a self-built knowledge based engine with something like Wolfram|Alpha's computational knowledge engine in a Unity3d.com game engine for example. Or, one could use it to illustrate health issues with possible solutions using mind mapping tools in a health portal. Actually, ones imagination is the limit. Naturally, this would drive your business opportunities. Think about it.

OK, I have a Master's degree in Computer Information Systems from Boston University. My entrepreneurial mind often gets me in trouble. But, I am working on different projects from Unity3d games to health and political science projects. I am presently only using qiqqa for my Phd studies in Political Science. I am using Unity for game projects on Apple and Android. I am using Drupal and Joomla for health, golf and the RISC.lu projects. However, I could easily see a broadening role for an information management service like qiqqa which allows its customers to build information specific DB services using qiqqa as a backend service provider.

Bye,
Paul Moody

    Paul Moody over 8 years ago
    In my humble opinion as an entrepreneurial businessman, Qiqqa must create an API sooner than later in order to survive. Qiqqa’s software can be rapidly recreated in Open Source. Its major market share is in the heart of the Open Source movement, the Universities. If the Universities and/or students do not have the APIs they desire, they will simply create an Open Source version of Qiqqa. And, they have the global Open Source resources including University resources to do so.

    Personally, I like Qiqqa. I have no interest in creating such an Open Source project. However, I have researched the idea as a result of the missing API. Based on existing available Open Source software and readily available Cloud computing resources, I feel Qiqqa had better move quickly. If I were Qiqqa, I would remove this strong incentive for an Open Source project.

    Qiqqa has a great service for a very reasonable price. However, it is still low hanging fruit for an Open Source project. I am of the opinion that unless something is broke, there is no reason to fix it. Qiqqa is not broke. Qiqqa is a great firm. It needs an API with an appropriate service charge which I hope it will have time to build. However I fear time is not on their side.

    Therefore, Qiqqa has to prioritize their programming capability against a large competitor which I refer to as the Open Source movement. Qiqqa is doing the hard work, they are thinking through the thought process. Unfortunately, their work can be easily copied. One should not forget, most Open Source programming projects copy someone else’s hard earned mental process. Again, I say most meaning not all. Therefore an Open Source project can quicklycopy something like Qiqqa. After all, we now understand from Qiqqa what needs to be done.

    Luis, you are right about the up-loaders. However, I feel there are many aspects missing from Qiqqa which could be added given the appropriate API and 3rd party programming resources. It is actually the additional resources Qiqqa needs but can’t afford which would give them the additional innovative functionalities they absolutely require in order to ultimately compete. No firm can stand alone. Otherwise, Microsoft, Oracle, Facebook, Twitter, etc. would never have invested major resources into their APIs.

    Qiqqa’s business fears are probably about losing to soon the control of the development process and the additional business opportunities. However, I believe they may have more to fear if they don't let the process run its course.

    What all startups have to ultimately learn, you have to let the baby walk. It may stumble or fall but there is no real alternative if you want the baby to run. Otherwise, the baby loses the race.

    Paul Moody over 8 years ago
    Hi Qiqqa,

    I have patiently been waiting for an API for over a year. I guess it is not coming. I will now start a European consortium to create a Qiqqa like equivalent supporting a frontend from something like Zotero, Mendeley, Jabref, etc. connected to something like PHPKB, KBPublisher with a Drupal/Joomla management scenario. Since PDF conversion can be distributed across the users, there are enough open-source software to handle PDFs,

    By the way, I think Qiqqa is great; however, I want to make content specific applications using a knowledge base thematic approach like Qiqqa using some natural language features. And my true desire was never to compete with Qiqqa; I just have similar KB requirements. And Actually, I didn't care if it was open source, chargeable, etc. so my initial ideas were to use Qiqqa once an API was available. However, if I have to build one, then the only cost-effective way to do this is to string open source server applications together on an Amazon cloud using open-source software and its support network to quickly have a working application. In principle, I am tired of waiting.

    If anyone is interested in the project, let me know.

    Best wishes,
    Paul Moody
