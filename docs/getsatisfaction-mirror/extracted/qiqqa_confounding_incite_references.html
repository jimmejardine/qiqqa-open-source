Qiqqa confounding InCite references

I've recently experienced Qiqqa confounding references. I was using the InCite feature to add references to a report and I noticed that Qiqqa would insert a reference to a paper different from the one that was selected in library. I want to insert a reference to paper A, but Qiqqa would insert a reference to paper B (even though paper A is selected in the library). This problem seems specific to these two PDFs. To Qiqqa both references look like the same paper, but they are not. I deleted the two problemantic PDFs and then re-imported them but that did not correct the problem. I suspect this problem could be related to the origin of the documents - these PDF's are articles or statements published on the web by various agencies (e.g. FDA, World health organization). A downloaded pdf is not available, so I convert the webpage into a PDF (maintaining text searchability). My question is - What can I do to alleviate this confusion? I aslo tried re-indexing my library but that didn't help. As these papers come from the web I have minimal bibtex metadata. Could this be related to how they are referenced? IF so, brings up a second question, what is the best way to cite a web article? Is it ok if my library references are of mixed types (articles, proceedings, book, booklet, etc).

Thanks for you help!
