henrik over 7 years ago

SHORT NAME for journals now supported, but how?

In the current version of QIQQA it says that "short name" for journals is supported by QIQQA. What does that mean, what is the name of the journal-short field, does QIQQA use an internal database to maintain both LONG and SHORT names for journals throughout the library ? Do I manually have to enter both short and long names? Does the sniffer have any capability to enter both short and long names for new entries? ETC.. Please provide some information, other than stating that short names are now supported.

    Take a look at this thread.

    The abbreviated name is in journal-iso. This is downloaded automatically from Pubmed. Don't know about other sources.

    henrik over 7 years ago
    Hi, ok.. I thought the qiqqa support went a little further.. My problem is that the library contain only long names (as imported e.g. by the sniffer from G scholar). It would be quite easy to make Qiqqa generate the ISO names automatically on the fly or as a batch process to all entries that do not have a journal-iso field.
    Manually adding the journal-iso field to my thousands of entries is a rather tedious task. I hope Qiqqa will address this issue.
    The long/short switch in the CLS definition now works well in InCite (as long as the journal-iso is defined in the database). Many thanks to the Qiqqa team for that!

    Qiqqa (Official Rep) over 6 years ago
    The latest v67 now allows you to use abbreviations, both by enabling a default set of abbreviations, and by allowing you to supply your own custom abbreviations.

    The custom abbreviations file should be of the form:
    journal name 1<tab>abbreviation 1
    journal name 2<tab>abbreviation 2
    journal name 3<tab>abbreviation 3

    Thanks for your support!
