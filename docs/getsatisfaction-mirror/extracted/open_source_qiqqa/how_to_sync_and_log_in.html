Open Source Qiqqa - How to sync and log in

Hi guys, I just updated to v.80 on my laptop and I wanna know how can I sync my new intranet libraries on my tablet using onedrive (or googledrive or dropbox).

After I installed the new version and opened it, it only gave me the option to "start qiqqa". So I can neither log in using my old username and password nor can I create a new login. What should I do?

Will I be able to sync my intranet libraries without an account? HOW??

thank you!

    Ger Hobbelt 9 months ago
    Hi,

    I may misunderstand what you're asking, so there's a question below as well, next to a bit of general explanation as I guess a few things about your question.

    Qiqqa Open Source (v80 and onwards) don't have a login functionality anymore as only Commercial Qiqqa (v79 and before) have Premium / PremiumPlus access to the Qiqqa Cloud store at http://www.qiqqa.com/Libraries.

    Hence if you want to synchronize your local copy with *those* you MUST run Qiqqa v79 (or older) to do so.

    "Migration" to Qiqqa Open Source then comes down to sync-ing using v79, so that your local copy is up to date with the Qiqqa Cloud remote store at http://www.qiqqa.com/Libraries (this is only important if you work on multiple machines and/or collaborate with others on the same Qiqqa Cloud Library, i.e. if your current machine is not the only machine/user which has access to each of your Qiqqa libraries.

    Then, once you are running Qiqqa Open Source (v80 or later), access to your http://www.qiqqa.com/Libraries commercial account is not possible any more (Jimme took that out when open sourcing Qiqqa - I understand there are legal issues around that part of the Qiqqa code, alas). v80 will "discover" your local copies and list them as "Legacy Web Library- CrypticGUIDCode" in your Qiqqa library list, where you should be able to continue using them, though SYNC functionality is gone. v82pre5 will (very probably) list the same 'web library' by its original name, but v82pre5 has another problem: see below.

    Meanwhile, you CAN create a new "Intranet Library" in Qiqqa which you point at a DropBox/GoogleDrive/OneDrive/etc directory as a sync point and then copy/move your documents to that new library.

    Cave Canem: v80 is v79 with Qiqqa Cloud store / Commercial Account Access removed; I published a few v82 prereleases (v81 was another release of mine, but that was VERY experimental and not recommended in any way) but haven't tested Sync functionality all that well while working on other issues -- I needed a Qiqqa which was able to survive a v79 library collection of mine which had somehow caused v79 and anything else to fatally crash, abort on search, etc., so that had top priority as I work alone on that library.

    Bottom line: v82pre5 and v82pre4 are getting there, but I managed to b0rk the sync functionality as reported here: https://github.com/jimmejardine/qiqqa... and here: https://github.com/jimmejardine/qiqqa... so you will have to wait a while for me to get that fixed and back up. Then there's also issue https://github.com/jimmejardine/qiqqa... and https://github.com/jimmejardine/qiqqa... which are relevant to sync functionality becoming really usable again.

    That's what I can say now, while I do a bit of handwaving 'cause I *may* misunderstand what you're asking.

    Two questions:

    1) you asked "how can I sync my new intranet libraries on my tablet" and that makes me wonder: does your tablet run Windows? (Because Qiqqa runs only on Windows machines. That's why *tablet* makes me go hmmmmmm)

    2) Much appreciated if you can file this question = issue at the github site, because that's the main dev platform now and the GetSatisfaction site is becoming a spamming target (sad face) AFAICT. Github is not just for developers and accounts are free. :-)

    HTH
