Link collection-based pdfs?

Is it possible to link documents? For instance, I have pdf scans of individual book chapters and I would like them to remain separate for both research and citational purposes but it would be great to be able to link them to other chapters in the book. This is equally true of articles in special issues, for instance. I'd like to see them as linked files on the side, much like you can see the relevant other pdfs but more explicitly listed. Tagging doesnt really do the trick.

Thanks!

    Make sure you are in Expert Mode (click ExpertMode? in the bottom left)
    Open a PDF 
    On the left there are some infobar tools.
    Expand the 'linked docs' section.
    Search for a 'target doc' and press ENTER to link it.

    Ger Hobbelt over 4 years ago
    Re 'mass linking': here's what I do as a work-around to 'bundle book chapters':

    (we assume you want to link several chapter PDFs of a single book)

    - select (find or via other means) all the chapters in your library

    - now mass-add (Tags::Add-Multiple) a unique tag identifying the book, e.g. 'book:SMPS-dev'

    - presto!

    Notes:

    - I use tags with a colon ':' for this purpose alone, creating tags which start with the *generic type* (e.g. 'book'), followed by the identifying (book) name after the colon as in the example above.

    - Now all this needs as an extra is tag wildcard/regex search, e.g. 'book:.*' to dig up all the (bundled) books from your library in case you want to go and merge those PDF files, fr example. ;-)
