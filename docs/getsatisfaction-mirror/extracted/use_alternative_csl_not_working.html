Use alternative CSL not working

I have been promoting Qiqqa with my colleagues. Much to my dismay they have experienced problems with incite.

They get a reproducible error using soil-biology-and-biochemistry.csl and forest-ecology-and-management.csl.

The error is as follows:

ERROR: Unable to find key e20cbf2c0ac0428cad8d4d3259fc459e in the CSL output.

References

Either this CSL citation style does not produce a bibliography section or you have not yet added any citations to this document, so this bibliography is empty.

    Dennis over 6 years ago
    soil-biology-and-biochemistry.csl is what is known as a "dependent style". It has no styling information of its own, only a pointer to another style with the styling information. In other words, it is an alias.

    I tried using the style and got the same error.

    Looking at the style in the style editor shows it references "elsevier-harvard.csl" as an "independent style".

    Tell your colleagues to try the associated "independent style" which in this case would be called "Elsevier-Harvard (with titles)" in the Zotero style repository. That one works fine in Qiqqa.

    It would be nice if Qiqqa could handle the dependent->independent style redirection automatically, or at least provide a less cryptic error message. In the meantime, this workaround should solve the problem.
